Usability Test Plan Section

Test Plan for: Product/Application Component

Persona Type Tested: Primary/Secondary/Negative/Customer

Context: an environment (location) where the user is comfortable without distractions
Duration: 18 minutes

Complete the following before starting: Greeting and Consent & User Background Information.

Overall Objectives
  • Evaluate the simplicity/efficiency of the application
  • Identify performance issues
  • Evaluate the accurancy/efficiency of scanning the qr codes
  • Identify difficulities in memorizing the commands structure
Scenario

The user needs to see a lsit of deals based on a qr code scanned and filter by categories and location.

Tasks

Write out the tasks for each persona type. Avoid complex tasks. Make sure the user is able to complete the test in the specified time interval. Assign an estimated completion time for each task.

  • Task 1: Install application with the specific name(4 minutes);
  • Task 2: Open application and visualize the list of deals(2 minute);
  • Task 3: Go on qr-code from menu (30 seconds);
  • Task 4: After the camera opens, scan qr-code(2 minutes);
  • Task 5: Visualize the list of deals filter by location obtained after scanning(1 minute);
  • Task 6: Write the name of a store in the search bar and visualize the result(2 minute);
  • Task 7: Click on one deal and visualize the details page with additional info(2 minutes);
  • Task 8: Click on the back button to return to the main page (30 seconds);
  • Task 9: Go on Categories menu item (1 minuute);
  • Task 10: Select the favourite categories (2 minutes);
  • Task 11: Return to the main page and visualize the deals filter by categories(1 minutes);
Post-test Questions
  • Question 1: How difficult it was to use the qr-code functionallity?(1-easy, 5-difficult)
  • Question 2: How difficult it was to walk through menu?(1-easy, 5-difficult)
  • Question 3: Would you recommend the application to your friends?(1-easy, 5-difficult)
  • Question 4: How fast did the application respond to your commands?(1-easy, 5-difficult)
  • Question 5: How fast did the deals filter?(1-easy, 5-difficult)
Notes for Test Coordinator
  • Remind the user that you would like them to 'think aloud' so that you can understand what they are thinking, about the task or the product/application;
  • Read the user the scenario for the test;
  • Read the user the tasks he has to complete. If the tasks are complex, you may want to put a sheet of paper in front of the user with each task written out;
  • Record the user's actions, comments, questions, and body language (write them in the Test Observations section accompanied by video/audio).
  • Don't offer help; let the user attempt to perform the task. If they ask for help reply with: "What do you think you/that would do?" "What do you think that means?".

Test Results for: Product/Application Component

Success Criteria

A successful design has been achieved when:

  • 75% of users complete successfully all tasks,
  • 50% of users manage to execute completed tasks in given time,
  • 75% of users have overall positive impression about application.
